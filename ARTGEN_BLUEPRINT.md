# Heroquarium ArtGen — блюпринт отдельного проекта

Документ самодостаточный: по нему можно поднять проект **с нуля в новой сессии**, не имея
контекста переписки. Цель — конвейер генерации пиксель-арта для игры Heroquarium: сначала
**процедурный** генератор (кодом), затем **нейросетевой** (Stable Diffusion на локальном GPU),
и **MCP-сервер**, через который ассистент (Claude) вызывает эти инструменты напрямую.

> Родительская игра: Heroquarium — zero-player top-down RPG (монорепо sim/server/client,
> Colyseus + Postgres + Pixi.js). Клиент рендерит мир тайлами **16×16** из `packages/client/src/assets/*.png`.
> Этот проект (`heroquarium-artgen`) — отдельный репозиторий, поставляющий ассеты в игру.

---

## 0. TL;DR — что строим

1. **procgen** — процедурный генератор тайлов и спрайтов (Python + Pillow). Даёт террейн,
   бесшовные autotile-переходы, воду с пеной, параметрические дома, заборы, деревья, камни,
   пропсы. Детерминированно, бесплатно, всегда в палитре игры, швы корректны по построению.
2. **pixelize** — нормализатор: любой внешний арт (нейронка/Kenney/Recraft) → 16 или 32 px
   в палитре игры, чистая альфа. Уже написан в игре (`packages/client/tools/pixelize.py`),
   переносим сюда как общий модуль.
3. **nngen** (позже) — генерация через ComfyUI/SDXL + пиксель-LoRA на GPU (RTX 3080 Ti),
   с дообучением LoRA на референсах. Выход прогоняется через pixelize.
4. **mcp** — MCP-сервер, экспонирующий все инструменты (генерация тайла, autotileset,
   здания, пикселизация, сборка атласа, превью) как вызываемые Claude-инструменты.
5. **Интеграция** — экспорт в формат ассетов Heroquarium + манифест, чтобы игра подхватывала.

Порядок работ: procgen → pixelize (перенос) → mcp (обёртка над ними) → nngen → интеграция.

## 1. Цели и не-цели

**Цели**
- Единый, консистентный набор ассетов в тёплом уютном стиле (Stardew/roguelike-town).
- Бесшовный террейн и переходы между биомами (autotile) — главная боль карты.
- Всё в канонной палитре игры (см. §4) и в целевом размере тайла.
- Повторяемость: один seed → тот же результат (для procgen).
- Доступность из Claude через MCP: «сгенерь набор травы с переходами на песок» → готовые PNG.

**Не-цели (пока)**
- Не рисуем богатую «киношную» иллюстрацию как в крупных концепт-референсах — это ориентир
  настроения, а не то, что кладётся в игру тайлами.
- Не делаем полноценный редактор тайлов (используем Aseprite для ручной доводки при нужде).
- Не встраиваем генерацию в рантайм игры — это оффлайн-пайплайн, продукт = файлы ассетов.

## 2. Стек и требования

- **Python 3.11+**, `Pillow`, `numpy`. (procgen, pixelize, mcp — всё на Python, чтобы один стек.)
- **MCP**: официальный Python SDK `mcp` (или `fastmcp`), транспорт **stdio** для локального
  запуска. См. §8.
- **nngen (GPU-машина)**: Windows + WSL2 (Ubuntu) ЛИБО нативный Windows; ComfyUI, PyTorch с CUDA,
  модель SDXL (base) + пиксель-арт LoRA; для тренировки — `kohya_ss` или `sd-scripts`. VRAM 12 ГБ
  (3080 Ti) достаточно для инференса SDXL и тренировки LoRA (batch 1–2, 512–768 px).
- ОС для procgen/mcp: любая (Linux/macOS/Windows). GPU не нужен.

## 4. Канон: размеры, палитра, соглашения

**Размер тайла.** Текущая игра — **16×16**. Целевой для нового арта — обсуждаемо: 16 (влезает
как есть) или **32×32** (детальнее; тогда рендер игры либо переводим на tile=32, либо крупные
пропсы рисуем больше клетки, как деревья сейчас). Генератор параметризуем по `TILE`; экспорт
умеет отдавать оба. **Решение по умолчанию: генерировать в 32, экспортировать даунскейлом в 16
для текущего рендера, хранить оба.**

**Палитра (53 цвета).** Единый источник — `palette.json`. Вытащена из текущих ассетов игры.
Всякий новый арт квантизуется к ней. Полный список — Приложение A. Ключевые семейства:
- Зелёные (трава/листва): `#345527 #2f5a2b #3e6631 #417a37 #4a7a3a #4f7a3a #446a32 #5f8f46 #4a8f4a #5aa843 #5fa85f #7cc45e`
- Земля/дерево (тёплые коричневые): `#3a2417 #3a2a1a #5a3a24 #6e4a2f #7a4a2a #7a5238 #8f4f2f #8f5a3a #94684a #b0623a #b55a33 #cc7444`
- Песок/пергамент: `#c4a86a #cbb08a #d8a878 #d9c07f #e3cf94 #e3d05a #e8b088 #e8c8a8`
- Вода/небо (синие): `#2f5e8f #3f75ad #4a5a8f #7fa8cc`
- Камень/UI-нейтраль: `#241d2b #3a3a44 #565061 #6a6475 #7a7485 #948da3 #a8a8a0 #c8c8c0 #c9c2b5 #e8e4dc #e8e8e0`
- Акценты (золото/кровь): `#e8c15a #ffd982 #b0453a #cc6452 #d97b7b`

**Прочие соглашения**
- Прозрачный фон (RGBA), альфа бинарная (порог 128) — без полупрозрачных краёв.
- Свет сверху-слева, лёгкое затенение снизу-справа (единое по всему набору).
- Тонкий тёмный контур у объектов (цвет из тёмной части палитры), у грунтов — без контура.
- top-down орто-проекция.
- Имена файлов — латиница, `snake_case`, по контракту игры (§9).

## 5. Компонент procgen — процедурный генератор

Идея: маленькие 16/32-пиксельные изображения строятся из (а) «пиксель-карт» (строки-символы
→ цвета палитры — как в текущем `gen_assets.py`), (б) алгоритмических заливок (value-noise,
ramps, дизеринг), (в) параметрических форм (дома, заборы). Всё детерминировано от seed.

### 5.1 Базовые примитивы (`canvas.py`)
- `img(TILE)` — пустой RGBA.
- `from_map(rows, pal)` — 16/32 строк по 16/32 символов → тайл (перенести из `gen_assets.py`).
- `ramp(c0, c1, n)` — линейная растяжка цвета (для мягкого затенения).
- `value_noise(w, h, seed, octaves)` — связный шум для крапа травы/песка.
- `dither(a, b, pattern)` — упорядоченный (Bayer 2×2/4×4) дизеринг двух цветов.
- `drop_shadow(sprite, dx, dy, color, alpha)` — мягкая тень под пропсом.
- `outline(sprite, color)` — 1px контур по альфа-краю.
- `mirror_x(sprite)` — симметрия для существ/объектов.
- `snap_palette(sprite, palette)` — финальная квантизация (использует `palette.py`).

### 5.2 Террейн (`terrain.py`)
Функции `grass(seed)`, `dirt`, `sand`, `stone`, `snow`, `forest_floor`, `swamp`, `water(frame)`.
Каждый грунт — базовый цвет + 2 оттенка крапом по value-noise, 2–4 варианта на биом (variety),
детерминированы seed. Вода — 2–3 кадра ряби для анимации (как `water0/water1`).

### 5.3 Autotile-переходы (`autotile.py`) — ключевое
Реализуем **blob/47-tile** (или проще — 16-tile 4-bit Wang по сторонам) для переходов
биом↔биом. Вход: два грунта (напр. трава→песок). Выход: набор краевых тайлов (углы, стороны,
внутренние углы), которые гарантированно стыкуются. Метод: маска формы края (кривая/грубая
органика по noise) + композитинг верхнего грунта над нижним по маске + общий контур кромки.
Тест `test_autotile.py`: для каждого варианта пиксели общей стороны совпадают с соседним
вариантом (кромки бесшовны).

> Порядок переходов, минимально нужный игре: grass↔dirt, grass↔sand, sand↔water,
> grass↔water(берег с пеной), grass↔stone, snow↔stone. Пена — отдельный оверлей на водной
> стороне берега (2 кадра).

### 5.4 Природа (`nature.py`)
Деревья (лиственное круглое, ель, палтма), кусты, пни, валуны, мелкие камни, цветочные
кластеры, грибы, кости/череп, тростник, тафты травы. Каждый — центрирован, с `drop_shadow`.
Деревья могут быть больше клетки (рисуются поверх, y-сортировка на стороне игры).

### 5.5 Деревня (`buildings.py`)
Параметрический дом: размеры (S/M/L), тип крыши (щипцовая/скатная), цвет крыши (сланец-синий,
красная черепица, солома), материал стен (камень/дерево/фахверк), дверь, 0–2 окна. Плюс: забор
+ калитка, колодец, бочки/ящики, вывеска, фонарь на столбе, рыночная палатка с полосатым
навесом, поленница, стог. Всё — из общей палитры и с единым светом.

### 5.6 Существа (`creatures.py`)
Простые силуэты мобов/NPC через симметрию + палитра. Для богатых существ — путь nngen.

### 5.7 CLI
```
python -m artgen.procgen terrain --tile 32 --seed 42 --variety 3 -o out/terrain/
python -m artgen.procgen autotile --from grass --to sand -o out/edges/
python -m artgen.procgen building --size M --roof red --walls timber -o out/house_m.png
python -m artgen.procgen all -o out/            # весь набор + атлас + манифест + превью
```
Каждая команда пишет PNG + строку в манифест (§9) и опционально контактный лист (§7 preview).

## 6. Компонент pixelize — нормализатор внешнего арта

Уже написан и проверен в игре: `packages/client/tools/pixelize.py`. **Скопировать в
`artgen/pixelize.py`** и импортировать палитру из общего `palette.json`. Возможности:
- вход — картинка или атлас; нарезка `--grid COLS ROWS` или `--src-tile N`;
- даунскейл (`box`/`lanczos` для «жирного» арта, `nearest` для уже-пиксельного) до TILE;
- квантизация к палитре по перцептивному расстоянию (redmean), бинарная альфа по порогу;
- `--rebuild-palette` — пересобрать палитру из эталонных ассетов.

Это единый «финишер» и для procgen (страховка), и для nngen (обязательный шаг), и для чужих
наборов (Kenney/Recraft). Интерфейс (важен для MCP §8): функции `load_palette()`, `quantize()`,
`downscale()`, `slice_tiles()`, `process(...)`.

## 7. Целевой стиль и промпты (ориентир для nngen и ручной генерации)

Стиль-эталон — тёплый уютный top-down (Stardew Valley × классический roguelike-town): слоёная
трава с тёмными пятнами, землистые тропы с галькой по краям, каменно-деревянные дома со сланцевой
черепицей, мягкое затенение, тёплая насыщенная палитра, лёгкий тёмный контур, свет сверху-слева.

Готовый набор промптов для тайлсетов (террейн, природа, деревня, вода/причалы, лагеря) лежит в
исходной игре в `references/recraft-tileset-prompts.md` — **перенести в `nngen/prompts.md`**.
Общий хвост к каждому промпту:
> top-down orthographic view, cohesive warm palette, soft shading, subtle dark outlines,
> cozy fantasy village RPG, crisp pixel art, neat aligned grid of separate tiles,
> transparent background, no text, no labels, consistent lighting from top-left.

`preview.py` даёт две проверки качества: **контактный лист** (до/после квантизации) и
**tileable-тест** (склейка тайла 3×3 и подсветка расхождений по кромкам).

## 8. Компонент mcp — доступ Claude к инструментам

MCP-сервер (Python, транспорт **stdio**) экспонирует генерацию как вызываемые инструменты.
Тогда в сессии можно просто просить «сгенерируй autotile трава→песок, вариаций 3» — и получать
файлы, без ручного запуска CLI.

### 8.1 Инструменты (первый срез)
- `list_palette()` → массив hex + tile size.
- `generate_terrain(kind, tile=32, seed=42, variety=3)` → пути к PNG + предпросмотр (base64).
- `generate_autotileset(from_kind, to_kind, tile=32, seed=42)` → набор краевых PNG + превью.
- `generate_building(size, roof, walls, tile=32, seed=42)` → PNG + превью.
- `generate_nature(kind, count=1, seed=42)` → PNG.
- `pixelize_image(path|base64, tile=16, method="box", grid=None, src_tile=None)` → нормализованный PNG.
- `build_atlas(dir, cols)` → атлас + манифест JSON.
- `preview_sheet(paths[])` → контактный лист (base64 PNG) для показа в чате.
- `tileable_check(path)` → склейка 3×3 + метрика шва.
- (позже) `nn_generate(prompt, n=4, size=768, lora=...)` → PNG через ComfyUI (см. §10).

Каждый инструмент возвращает и путь(и) на диск, и **base64-превью**, чтобы Claude сразу
показывал результат в чате.

### 8.2 Каркас сервера (эскиз)
```python
# mcp/server.py
from mcp.server.fastmcp import FastMCP
from artgen import procgen, pixelize, atlas, preview, palette

app = FastMCP("heroquarium-artgen")

@app.tool()
def list_palette() -> dict:
    cols, tile = palette.load()
    return {"tile": tile, "colors": cols}

@app.tool()
def generate_autotileset(from_kind: str, to_kind: str, tile: int = 32, seed: int = 42) -> dict:
    paths = procgen.autotile.generate(from_kind, to_kind, tile, seed, out="out/edges")
    return {"paths": paths, "preview_png_b64": preview.sheet_b64(paths)}

# ... остальные инструменты аналогично

if __name__ == "__main__":
    app.run()  # stdio
```

### 8.3 Подключение к Claude
- **Локально (та же машина, что и Claude Code):** зарегистрировать stdio-сервер —
  `claude mcp add heroquarium-artgen -- python /path/to/mcp/server.py`
  (или соответствующий пункт в настройках коннекторов клиента). После этого инструменты
  `mcp__heroquarium-artgen__*` доступны в сессии.
- **На GPU-машине (см. §10):** MCP-сервер запускается ТАМ, где стоит ComfyUI, и включает
  `nn_generate`. Доступ к нему из клиента — через приватную сеть **Tailscale** (не публичный
  ngrok). Важно: смогут ли конкретные сессии дотянуться до удалённого MCP — зависит от сетевых
  политик коннекторов; надёжнее всего запускать агента/Claude Code прямо на GPU-машине.

### 8.4 Безопасность
- Инструменты пишут только в `out/` и явно указанные пути; не принимают произвольный shell.
- Удалённый MCP — только внутри Tailscale/приватной сети, с токеном; не выставлять наружу.

## 9. Интеграция с игрой Heroquarium (контракт)

Клиент грузит ассеты так (`packages/client/src/render/renderer.ts`):
```ts
const ASSET_URLS = import.meta.glob('../assets/*.png', { eager, query:'?url' });
// name -> ../assets/${name}.png, scaleMode = 'nearest', tile 16×16
```
Значит **контракт экспорта**: PNG с точными именами в `src/assets/`. Текущий инвентарь имён
(минимум, который надо покрыть, чтобы ничего не сломать):
```
grass0 grass1 grass_flowers forest_floor sand0 sand1 water0 water1 rock_floor
tree cactus rock house tent hero trader
mob_slime mob_boar mob_scorpion mob_skeleton
```
`export.py`:
- принимает сгенерированные тайлы (32 px) → даунскейл в 16 px под текущий рендер (или оставляет
  32, если игру переведут на tile=32);
- раскладывает под каноничные имена + новые (autotile-края, здания, пропсы);
- пишет `manifest.json` (имя → тип, размер, seed, происхождение procgen/nn) для трассируемости;
- НЕ коммитит в игру сам — отдаёт папку, перенос в игру — отдельный осознанный шаг.

> Новое (autotile-переходы, крупные здания) потребует правок на стороне игры: рендер грунтов
> должен уметь выбирать краевой тайл по соседям (bitmask), а `sim`/worldgen — отдавать биом
> соседних тайлов. Это будущая задача уже в репозитории игры, не в artgen. Здесь — только ассеты
> и манифест с описанием, какие индексы краёв за что отвечают (документировать в manifest).

## 10. Компонент nngen — генерация на GPU (этап 2)

Железо: Windows + RTX 3080 Ti (12 ГБ). Рекомендуется **WSL2 (Ubuntu)** для чистого CUDA-стека,
либо нативный Windows.

1. **ComfyUI** + PyTorch/CUDA. Модель: **SDXL base** (или SD1.5 для скорости) + **пиксель-арт
   LoRA** (публичные на Civitai) как старт.
2. **generate.py** — клиент ComfyUI HTTP API: принимает промпт/размер/seed/LoRA, кладёт воркфлоу,
   забирает PNG. Далее **обязательно** через `pixelize` → палитра/размер игры.
3. **Своя LoRA под стиль референсов** (даёт попадание в уютный стиль): датасет 20–60 кропов из
   `references/` + подписи; тренировка `kohya_ss`/`sd-scripts`, 768 px, batch 1–2, ~1500–3000
   шагов. Результат — `heroquarium_style.safetensors`, подключается в ComfyUI.
4. Роль в MCP: инструмент `nn_generate(prompt, n, size, lora)` — но только на MCP-сервере,
   запущенном на GPU-машине (§8.3).

Пайплайн целиком: `nn_generate` → `pixelize` → `build_atlas` → `export` → (ручная проверка) → игра.

## 11. Дорожная карта (вехи)

1. **M1 — каркас + палитра + pixelize.** Репо, `pyproject`, `palette.json`, перенос `pixelize.py`,
   тесты палитры. (полдня)
2. **M2 — террейн + природа.** `terrain.py`, `nature.py`, вариации, контактные листы. (1–2 дня)
3. **M3 — autotile.** Переходы grass/dirt/sand/water + пена, tileable-тесты. (2–3 дня, ядро)
4. **M4 — деревня.** `buildings.py` параметрически + пропсы. (1–2 дня)
5. **M5 — MCP-сервер.** Обёртка инструментов, base64-превью, подключение к Claude. (1 день)
6. **M6 — экспорт + манифест + первый занос в игру** (заменить процедурные тайлы спрайтами). (1 день)
7. **M7 — nngen.** ComfyUI на 3080 Ti, generate.py, старт на публичной LoRA. (1–2 дня)
8. **M8 — своя LoRA** на референсах + инструмент `nn_generate` в MCP. (2–3 дня)

## 12. Критерии приёмки / планка качества

- Любой сгенерированный ассет — строго в палитре (`test_palette`: 0 цветов вне палитры).
- Autotile-варианты бесшовны (`test_seamless`/`test_autotile`: расхождение по общей кромке = 0).
- Один seed → идентичный выход (детерминизм procgen).
- Контактный лист набора визуально консистентен (единый свет/палитра).
- Экспорт покрывает все имена из контракта §9, игра запускается без отсутствующих ассетов.
- MCP-инструменты возвращают и файл, и превью; ошибки — понятным сообщением, без падения сервера.
